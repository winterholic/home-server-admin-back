# 홈서버 대시보드 디자인 가이드

## 전체 디자인 컨셉

**분위기**: 다크 모드 베이스의 모던하고 프로페셔널한 시스템 모니터링 도구
**색상 테마**:
- 배경: 매우 어두운 네이비/블랙 계열 (#0a0e27, #1a1f3a 같은)
- 강조색: 네온 블루, 사이버틱한 민트색 (#00f3ff, #6ee7b7)
- 경고색: 앰버/오렌지 (#fbbf24), 위험색: 레드 (#ef4444)
- 텍스트: 화이트/연한 그레이 계열

**전체 레이아웃 구조**:
왼쪽에 사이드바 네비게이션, 오른쪽에 메인 컨텐츠 영역. 사이드바는 좁고 미니멀하게(아이콘+텍스트), 메인 영역은 넓게 활용. 상단에는 현재 시간, 서버 업타임, 사용자 프로필 같은 글로벌 정보바.

**디자인 스타일**:
- 카드 기반 레이아웃 (각 카드는 미묘한 그림자와 테두리 글로우 효과)
- 유리 모피즘(Glassmorphism) 느낌 살짝 가미 (카드에 반투명 배경, 블러 효과)
- 데이터 시각화는 선명하고 가독성 높게
- 부드러운 애니메이션 (호버, 전환 효과)

## 1. 대시보드 메인 페이지

### 페이지 목적
서버의 전체 상태를 한눈에 파악할 수 있는 홈 화면. 사용자가 가장 자주 확인하는 핵심 지표들을 요약해서 보여줌.

### 주요 기능
- 실시간 시스템 리소스 사용률 (CPU, 메모리, 디스크, 네트워크)
- 서비스 상태 요약
- 최근 알림 3개
- 최근 중요 로그 이벤트 5개

### 레이아웃 구성

**상단 영역** (전체 너비):
4개의 메트릭 카드를 가로로 배치. 각 카드는 동일한 크기로 그리드 형태.

**카드 1: CPU 사용률**
- 왼쪽에 큰 숫자로 현재 사용률 표시 (예: "45.2%")
- 오른쪽에 작은 미니 라인 차트 (최근 1시간 추이)
- 카드 배경색은 사용률에 따라 살짝 변화 (낮음: 블루, 보통: 민트, 높음: 오렌지, 위험: 레드)
- 카드 상단에 작은 CPU 아이콘과 "CPU" 라벨

**카드 2: 메모리 사용률**
- 동일한 구조로 메모리 사용률 표시
- 프로그레스 바 형태로도 표현 (전체 메모리 대비 사용량)
- 하단에 작은 텍스트로 "16.1GB / 32GB" 같이 구체적 수치

**카드 3: 디스크 사용률**
- 동일한 구조
- 여러 파티션이 있으면 메인 파티션(/)만 표시
- 원형 프로그레스 바 형태도 괜찮음

**카드 4: 네트워크**
- 업로드/다운로드 속도를 화살표로 표시
- 업↑: 초록색, 다운↓: 파란색
- 실시간 전송량 표시 (예: ↑ 1.2 MB/s, ↓ 3.4 MB/s)

**중단 영역** (2열 그리드):

**왼쪽 (60% 너비): 서비스 상태 카드**
- 제목: "Services Status"
- 서비스 목록을 세로로 나열 (nginx, mariadb, redis, 앱들)
- 각 서비스는 한 줄로 표시:
    - 왼쪽: 서비스 이름
    - 가운데: 상태 뱃지 (Active=초록 점, Inactive=회색 점, Failed=빨강 점)
    - 오른쪽: 간단한 정보 (업타임 또는 메모리 사용량)
- 리스트 형태로 깔끔하게, 호버하면 살짝 하이라이트

**오른쪽 (40% 너비): 최근 알림**
- 제목: "Recent Alerts"
- 최근 알림 3개를 카드 형태로 표시
- 각 알림은:
    - 왼쪽에 경고/에러 아이콘 (심각도에 따라 색상 다르게)
    - 알림 메시지 텍스트
    - 하단에 시간 표시 (예: "2 hours ago")
- 알림이 없으면 "No recent alerts" 메시지와 체크 아이콘

**하단 영역** (전체 너비):

**최근 로그 이벤트 카드**
- 제목: "Recent Log Events"
- 테이블 형태로 표시:
    - 컬럼: 시간 | 타입 | 소스 | 메시지
    - 각 로그 타입에 따라 다른 색상 뱃지 (브루트포스=빨강, 에러=오렌지, 정보=파랑)
- 최대 5개 항목, 스크롤 가능
- 전체 로그 보기 버튼 (오른쪽 상단)

### 디자인 디테일

- **카드 스타일**: 배경 #1a1f3a, 테두리 1px solid rgba(100,200,255,0.1), border-radius 12px, 미묘한 그림자
- **숫자 폰트**: 큰 숫자는 모노스페이스 폰트(JetBrains Mono, SF Mono 같은) 사용해서 시스템 도구 느낌
- **차트 색상**: 라인 차트는 그라데이션 활용 (위쪽 밝은 블루 → 아래쪽 투명)
- **간격**: 카드 사이 여백은 24px, 섹션 사이는 40px
- **애니메이션**: 실시간 데이터가 업데이트될 때 숫자가 부드럽게 카운팅 애니메이션

## 2. 모니터링 페이지

### 페이지 목적
시스템 리소스 사용량을 시간대별로 상세히 분석할 수 있는 페이지. 과거 데이터를 차트로 시각화.

### 주요 기능
- 실시간 메트릭 그래프
- 시간대 선택 (1시간, 24시간, 7일, 30일)
- CPU/메모리/디스크/네트워크 각각 상세 차트
- 코어별 CPU 사용률 확인

### 레이아웃 구성

**상단 컨트롤 바**:
- 왼쪽: 페이지 타이틀 "System Monitoring"
- 오른쪽: 시간 범위 선택 버튼 그룹 (탭 형태)
    - [1H] [24H] [7D] [30D]
    - 선택된 탭은 네온 블루 배경, 나머지는 투명

**메인 차트 영역** (세로로 4개 카드):

**카드 1: CPU 모니터링**
- 제목: "CPU Usage"
- 큰 라인 차트 (시간별 CPU 사용률)
    - X축: 시간, Y축: 사용률(0-100%)
    - 라인 색상: 밝은 블루, 그라데이션 fill
    - 그리드 라인은 매우 연하게
- 오른쪽 사이드: 현재 CPU 정보
    - 현재 사용률 큰 숫자
    - 평균/최고/최저 사용률 (작은 텍스트)
    - 코어별 사용률 바 차트 (작은 크기로 4-8개 코어 표시)

**카드 2: 메모리 모니터링**
- 제목: "Memory Usage"
- 에어리어 차트 (RAM + Swap 구분)
    - 두 가지 색상 레이어로 표시
    - RAM: 파란색 계열
    - Swap: 보라색 계열
- 오른쪽 사이드: 메모리 통계
    - 현재 사용량 / 전체 용량
    - 사용 가능 메모리
    - 스왑 사용량

**카드 3: 디스크 모니터링**
- 제목: "Disk Usage"
- 파티션별로 탭으로 구분하거나 여러 라인으로 표시
- 각 파티션 색상 다르게
- 오른쪽 사이드: 파티션 리스트와 사용률

**카드 4: 네트워크 모니터링**
- 제목: "Network Traffic"
- 듀얼 라인 차트 (업로드/다운로드)
    - 업로드: 초록색 라인
    - 다운로드: 파란색 라인
- 오른쪽 사이드: 총 전송량, 평균 속도

### 디자인 디테일

- **차트 스타일**:
    - 배경 투명 또는 매우 어두운 그라데이션
    - 그리드는 점선, 색상 rgba(255,255,255,0.05)
    - 툴팁은 유리 모피즘 스타일 (블러 배경)
    - 차트 위에 마우스 호버하면 세로 가이드 라인 표시
- **반응형**: 작은 화면에서는 사이드 정보가 차트 하단으로 이동
- **데이터 포인트**: 차트에서 중요한 피크 지점에 작은 점 표시

## 3. 서비스 관리 페이지

### 페이지 목적
systemd 서비스들의 상태를 확인하고 제어할 수 있는 관리 페이지.

### 주요 기능
- 서비스 목록 테이블
- 각 서비스 상태 확인
- 서비스 시작/중지/재시작 버튼
- 서비스별 간단한 로그 보기

### 레이아웃 구성

**상단 헤더**:
- 왼쪽: "Service Management" 타이틀
- 오른쪽:
    - 새로고침 버튼 (아이콘만, 회전 애니메이션)
    - 서비스 검색 입력창

**메인 테이블 카드**:
큰 카드 하나에 테이블이 들어있는 형태

**테이블 구조**:
| 컬럼 | 설명 |
|------|------|
| Service | 서비스 이름 (아이콘 + 텍스트) |
| Status | 상태 뱃지 |
| Uptime | 가동 시간 |
| Memory | 메모리 사용량 (프로그레스 바) |
| Actions | 제어 버튼 |

**각 행 디자인**:
- 행 배경: 기본 투명, 호버시 살짝 밝은 배경
- 서비스 이름:
    - 왼쪽에 서비스별 아이콘 (nginx=N, mariadb=DB, redis=R 같은 심플한 아이콘)
    - 폰트 미디엄 웨이트
- 상태 뱃지:
    - Active: 초록 배경 + 체크 아이콘
    - Inactive: 회색 배경 + 일시정지 아이콘
    - Failed: 빨강 배경 + X 아이콘
    - 둥근 모서리, 작은 크기
- Uptime:
    - "3 days 2 hours" 형태
    - 회색 텍스트
- Memory:
    - 작은 프로그레스 바 (너비 100px 정도)
    - 옆에 숫자 표시 "52MB"
- Actions:
    - 아이콘 버튼 3개 가로 배치
    - ▶ (시작), ⏹ (중지), ⟳ (재시작)
    - 버튼 호버시 툴팁 표시
    - 위험한 작업(중지, 재시작)은 클릭시 확인 모달 띄우기

**서비스 상세 패널** (선택적):
- 테이블 행 클릭하면 하단에 확장 패널 나타남
- 패널 내용:
    - 최근 로그 10줄
    - 서비스 설명
    - 상세 메모리/CPU 정보

### 디자인 디테일

- **테이블 헤더**: 고정, 약간 어두운 배경, 작은 폰트
- **구분선**: 각 행 사이 매우 연한 회색 선
- **버튼**:
    - 기본 상태: 투명 배경, 테두리만
    - 호버: 해당 색상 배경 (시작=초록, 중지=빨강, 재시작=블루)
    - 클릭: 스케일 애니메이션
- **모달**:
    - 중앙 정렬, 어두운 오버레이
    - 유리 모피즘 스타일
    - "정말 nginx를 재시작하시겠습니까?" 메시지
    - Cancel / Confirm 버튼

## 4. 로그 뷰어 페이지

### 페이지 목적
시스템 로그, 보안 이벤트, 에러 로그를 조회하고 분석하는 페이지.

### 주요 기능
- 로그 타입별 필터링
- 시간대별 로그 통계
- 브루트포스 시도 감지 히스토리
- 로그 검색

### 레이아웃 구성

**왼쪽 사이드바 필터 패널** (30% 너비):

**필터 섹션**:
- "Filters" 타이틀
- 로그 타입 체크박스:
    - [ ] Brute Force Attempts
    - [ ] System Errors
    - [ ] Application Errors
    - [ ] Access Logs
    - [ ] All
- 소스 드롭다운:
    - Nginx, SSH, System, Applications
- 심각도 선택:
    - Info, Warning, Error
- 시간 범위 선택:
    - 라디오 버튼: Last Hour, 24 Hours, 7 Days, Custom
    - Custom 선택시 날짜 피커 표시

**통계 카드**:
- "Log Statistics" 소제목
- 작은 카드 3개 세로 배치:
    - "브루트포스 시도": 큰 숫자 + 빨강 아이콘
    - "에러 발생": 큰 숫자 + 오렌지 아이콘
    - "총 로그 수": 큰 숫자 + 파랑 아이콘

**오른쪽 메인 영역** (70% 너비):

**상단 컨트롤 바**:
- 왼쪽: "Log Viewer" 타이틀
- 오른쪽:
    - 검색 입력창 (아이콘 포함)
    - Export 버튼 (CSV 다운로드)

**로그 타임라인 차트** (상단):
- 작은 영역 차트
- X축: 시간, Y축: 로그 발생 빈도
- 로그 타입별로 색상 구분해서 스택형 차트
- 클릭하면 해당 시간대 로그로 필터링

**로그 리스트 카드**:
타임라인 형태로 표시하거나 테이블 형태

**타임라인 스타일**:
- 왼쪽에 세로 타임라인 바 (회색 선)
- 각 로그는 타임라인 노드
- 로그 카드 형태:
    - 왼쪽: 시간 + 타입 뱃지
    - 중앙: 로그 메시지 (주요 내용만, 길면 말줄임)
    - 오른쪽: 소스 아이콘
    - 클릭하면 펼쳐져서 전체 메시지 표시

**브루트포스 특별 표시**:
- 브루트포스 로그는 빨강 강조
- IP 주소 하이라이트
- "Block IP" 버튼 (선택사항)

**테이블 스타일 (대안)**:
| Time | Type | Source | Message | IP |
- 각 행은 타입에 따라 왼쪽 테두리 색상 다름
- 에러는 왼쪽에 빨강 두꺼운 테두리
- 경고는 오렌지, 정보는 파랑

### 디자인 디테일

- **필터 패널**:
    - 고정 위치 (스크롤해도 따라옴)
    - 배경 약간 어두운 카드
    - 체크박스는 커스텀 디자인 (체크시 네온 블루)
- **로그 메시지 폰트**:
    - 모노스페이스 폰트 사용
    - 작은 폰트 크기
- **심각도 색상**:
    - Error: #ef4444 (빨강)
    - Warning: #f59e0b (오렌지)
    - Info: #3b82f6 (파랑)
- **검색 하이라이트**:
    - 검색어와 일치하는 텍스트는 노란색 배경
- **무한 스크롤**:
    - 하단 도달시 자동으로 더 로드
    - 로딩 인디케이터 표시

## 5. 설정 페이지

### 페이지 목적
알림 설정, 이메일 설정, 모니터링 옵션을 관리하는 페이지.

### 주요 기능
- 알림 임계값 설정
- 이메일 수신자 관리
- 모니터링 수집 주기 설정
- 데이터 보관 정책

### 레이아웃 구성

**탭 네비게이션** (상단):
- [Alerts] [Email] [Monitoring] [Data]
- 선택된 탭 하단에 언더라인

**[Alerts] 탭 내용**:

**알림 설정 카드들** (각 메트릭별로 카드):

**CPU 알림 설정 카드**:
- 왼쪽 영역:
    - "CPU Usage Alert" 제목
    - 토글 스위치 (Enable/Disable)
- 오른쪽 영역:
    - "Threshold" 슬라이더
        - 0-100% 범위
        - 현재 값 표시 (예: 80%)
        - 슬라이더 색상: 값에 따라 그라데이션
    - "Alert when CPU exceeds 80%" 설명 텍스트

**메모리 알림 설정 카드**:
- 동일한 구조

**디스크 알림 설정 카드**:
- 동일한 구조

**서비스 다운 알림 카드**:
- 토글만 있고 임계값은 없음
- "Alert when any service goes down"

**보안 알림 카드**:
- "Brute Force Detection Alert"
- 토글 + 설정
    - "Failed attempts threshold": 입력 필드 (기본값 5)
    - "Time window": 드롭다운 (1분, 5분, 15분, 30분)

**[Email] 탭 내용**:

**SMTP 설정 카드**:
- 입력 필드들:
    - SMTP Server (예: smtp.gmail.com)
    - Port (예: 587)
    - Username (이메일 주소)
    - Password (비밀번호, 마스킹)
    - Use TLS (체크박스)
- "Test Connection" 버튼
- Save 버튼

**수신자 관리 카드**:
- "Email Recipients" 제목
- 이메일 주소 리스트:
    - 각 이메일은 작은 태그 형태
    - X 버튼으로 삭제 가능
- 새 이메일 추가 입력창 + "Add" 버튼

**[Monitoring] 탭 내용**:

**수집 주기 설정 카드**:
- "Data Collection Interval" 제목
- 라디오 버튼 선택:
    - ( ) 1 minute
    - (•) 5 minutes (권장)
    - ( ) 10 minutes
    - ( ) 15 minutes
- 설명: "더 짧은 주기는 더 많은 CPU를 사용합니다"

**모니터링 대상 설정 카드**:
- "Services to Monitor" 제목
- 체크박스 리스트:
    - [✓] Nginx
    - [✓] MariaDB
    - [✓] Redis
    - [✓] Custom Services

**[Data] 탭 내용**:

**보관 정책 카드**:
- "Data Retention Policy" 제목
- 슬라이더 또는 입력:
    - "Keep monitoring data for: 30 days"
    - "Keep log history for: 90 days"
- 경고 메시지: "오래된 데이터는 자동 삭제됩니다"

**데이터베이스 관리 카드**:
- 현재 DB 크기 표시
- "Vacuum Database" 버튼 (최적화)
- "Export All Data" 버튼
- "Clear All Data" 버튼 (위험, 빨강)

**하단 액션 버튼**:
- 오른쪽 하단에 고정
- "Cancel" 버튼 (투명 배경)
- "Save Changes" 버튼 (네온 블루 배경)

### 디자인 디테일

- **토글 스위치**:
    - Off: 회색, On: 네온 블루
    - 부드러운 전환 애니메이션
    - 스위치 내부에 작은 원
- **슬라이더**:
    - 트랙: 어두운 배경
    - 썸: 밝은 블루 원형
    - 채워진 부분: 그라데이션 (블루 → 민트)
- **입력 필드**:
    - 어두운 배경, 밝은 테두리
    - 포커스시 네온 블루 테두리 글로우
    - 플레이스홀더 회색 텍스트
- **버튼 스타일**:
    - Primary: 네온 블루 배경, 화이트 텍스트
    - Secondary: 투명 배경, 블루 테두리
    - Danger: 빨강 배경
    - 호버시 밝아지는 효과
- **카드 간격**: 각 카드 사이 24px

    - 서버 이름
    - 업타임 (작게)

**스타일**:
- 배경: 매우 어두운 색 (#0a0e27)
- 선택된 메뉴: 왼쪽에 네온 블루 세로 바, 배경 살짝 밝음
- 호버: 배경 약간 밝아짐
- 아이콘 + 텍스트 가로 배치
- 아이콘은 선 스타일 (outline)

### 상단 글로벌 바

**구조**:
- 전체 너비, 높이 60-70px
- 왼쪽: 빈 공간 또는 페이지 제목
- 오른쪽:
    - 현재 시간 (실시간 업데이트)
    - 알림 아이콘 (빨간 점으로 새 알림 표시)
    - 사용자 프로필 (드롭다운)

**스타일**:
- 배경: 투명 또는 매우 어두운 색
- 하단에 1px 얇은 테두리
- 고정 위치 (스크롤해도 상단 유지)

### 모달/다이얼로그

**스타일**:
- 배경: 어두운 오버레이 (rgba(0,0,0,0.8))
- 모달 박스:
    - 유리 모피즘 (블러 배경)
    - 둥근 모서리 16px
    - 그림자 강하게
- 헤더: 타이틀 + X 닫기 버튼
- 내용: 패딩 충분히
- 하단: 버튼들 (Cancel, Confirm)

### 로딩/스켈레톤

**스타일**:
- 스켈레톤: 회색 바 형태, 펄스 애니메이션
- 로딩 스피너: 네온 블루 원형 스피너
- "Loading..." 텍스트 아래 표시

### 애니메이션

- **페이지 전환**: 페이드인, 약간 위로 슬라이드
- **데이터 업데이트**: 숫자 카운팅 애니메이션
- **차트**: 로드시 라인이 왼쪽에서 오른쪽으로 그려지는 효과
- **호버**: 스케일 1.02, 그림자 강해짐
- **모든 애니메이션**: ease-in-out, 200-300ms

### 반응형

- **데스크탑**: 사이드바 고정
- **태블릿**: 사이드바 축소 (아이콘만)
- **모바일**: 사이드바 햄버거 메뉴로 토글
